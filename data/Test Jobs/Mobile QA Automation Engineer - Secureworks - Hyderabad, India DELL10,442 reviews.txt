Role Overview
We are looking for a Mobile QA Automation Engineer who will be responsible for planning, automating and implementing of tests that prove the functional and non-functional requirements of multi-platform mobile apps (iOS and Android) for next generation security analytics platform and incident response powered by AI & Machine learning, sustain & enhance automation for existing mobile application that keep the most important current and historical information about the company’s security posture at fingertips of customers using tools like Appium.
Role Responsibilities
Writes, executes test plans & report issues in Bug tracking system (preferably Jira, Qtest) for hybrid mobile applications that works on iOS & Android platforms.
Contributes to test automation for web, mobile applications using Appium & RESTful web services automation using Java/Python.
Test RESTful webservices that connects iOS/Android applications to back-end services.
Identifies performance bottlenecks & security related issues.
Minimum Requirements
Minimum 3 years of experience in writing & executing test cases related to UI/UX of mobile apps.
Minimum 2 years of experience in testing mobile applications backed integration with RESTful web services, Database/SQL etc.
Minimum 1 years of experience in automation using Appium.
Minimum 1 years of experience in backend RESTful services automation.
Preferred Skills
Familiarity with HTML 5, CSS, Java Scripts and web debugging tools
Exposure to AWS Services
